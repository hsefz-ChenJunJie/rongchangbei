# User Profile页面构建
## 用户个人页面设计方案

### 一、页面核心目标
1. **个性化适配** - 让AI更贴合用户的表达习惯
2. **效率提升** - 减少重复输入，提高交流效率
3. **隐私保护** - 安全存储敏感的个人信息
4. **易用性** - 考虑言语障碍用户的操作便利性

### 二、页面布局与结构

#### 顶部导航区
- 用户头像/标识
- 账户状态指示器
- 返回主界面按钮
- 页面标题“我的个人资料”

#### 左侧功能导航栏（垂直）
1. **基本资料**
2. **语料库管理**
3. **交流偏好设置**
4. **禁忌与敏感设置**
5. **设备与连接**
6. **数据备份与同步**
7. **帮助与反馈**

### 三、各功能模块详细设计

#### 模块1：基本资料
- **显示名称**（可编辑，支持emoji）
- **头像**（可上传图片或选择预设图标）
- **语言障碍类型**（可选：表达性障碍、理解性障碍、混合型等）
- **沟通辅助需求等级**（滑块：轻度/中度/重度）
- **常用交流场景**（多选：家庭、工作、医疗、社交等）
- **自我介绍模板**（预置一段自我介绍，便于快速向他人说明情况）

#### 模块2：语料库管理（核心功能）
采用标签化组织方式：

**2.1 常用短语库**
- 分类标签：问候、饮食、健康、紧急求助等
- 每条短语包含：
  - 短语内容
  - 使用频率统计
  - 快捷调用按钮
  - 可录制或输入替代发音（针对发音困难）

**2.2 个性化表达习惯**
- **句式偏好**：喜欢长句/短句/关键词式表达
- **语气词收集**：用户常用的语气词、连接词
- **缩写习惯**：个人常用的缩写（如“BRB”、“TTYL”等）
- **错别字纠正映射**：将用户常打错的字与正确字建立关联

**2.3 语境模板**
- 为不同场景预设表达框架：
  ```
  [医疗场景模板]
  主诉：{用户填写}
  病史：{自动填充常用病史}
  需求：{选择：开药/检查/复诊}
  ```

#### 模块3：交流偏好设置
**3.1 AI回复偏好**
- 理解耐心度：AI需要重复确认的程度
- 回复详细度：详细解释/简洁回答
- 建议主动性：主动建议/等待询问

**3.2 界面显示偏好**
- 文字大小
- 配色方案（高对比度模式）
- 是否显示预测文本
- 语音播报速度

#### 模块4：禁忌与敏感设置
**4.1 话题黑名单**
- 添加用户不愿涉及的话题（如某些疾病、经历等）
- 设置敏感词替代方案（当他人提及时，AI如何委婉处理）

**4.2 交流红线**
- 不能接受的表达方式（如命令式语气、过度同情等）
- 紧急回避词（听到特定词汇时自动转移话题）

#### 模块5：设备与连接
- 连接的辅助设备列表（语音设备、眼动仪等）
- 常用联系人管理（家人、医生、同事等及其沟通偏好）
- 跨设备同步设置

#### 模块6：数据管理
- **一键备份**到云端
- **数据导出**（支持JSON、TXT格式）
- **隐私模式**：临时禁用某些个性化设置
- **数据统计面板**：
  - 最常用短语TOP10
  - 交流成功率统计
  - 学习进度（新增掌握的词汇/表达）

#### 模块7：学习与改进
- **模型训练反馈**：对AI的建议进行“有帮助/无帮助”评分
- **表达困难记录**：记录交流失败的场景，用于优化
- **新词汇学习区**：添加最近想学习使用的新词汇

### 四、交互设计要点

#### 无障碍特性
1. **多重输入支持**
   - 语音输入（针对打字困难）
   - 图片选择（用图标表达）
   - 简化键盘布局
   - 眼动追踪支持

2. **操作便利性**
   - 大按钮设计
   - 重要操作二次确认
   - 操作历史记录和撤销功能
   - 语音导航支持

3. **紧急情况快速通道**
   - 页面任何位置可通过手势或快捷键触发紧急求助
   - 预设紧急联系人和信息

#### 视觉设计原则
- 高对比度配色方案
- 清晰的层级区分
- 重要信息图标化
- 动态效果克制（避免干扰）

### 六、安全与隐私考虑
1. **本地存储优先**：敏感信息默认本地存储
2. **选择性同步**：用户可自主选择同步到云端的项目
3. **加密存储**：所有个人语料加密存储
4. **数据使用透明度**：明确展示哪些数据用于模型训练
5. **一键清除**：快速清除所有个性化数据

---

### 七、扩展性设计
1. **API接口**：未来可连接其他辅助应用
2. **家庭成员管理**：家人可协助维护语料库（需用户授权）
3. **治疗师视图**：专业治疗师可查看进步报告（隐私隔离）
4. **社区词库**：匿名分享非敏感的表达方式

---

### 八、下一步开发建议
1. **MVP版本**：先实现基本资料、常用短语库、简单偏好设置
2. **用户测试**：优先邀请言语障碍用户进行可用性测试
3. **渐进增强**：根据用户反馈逐步添加高级功能
4. **多语言支持**：考虑不同方言和外语用户的需求

这个设计方案以用户为中心，特别考虑了言语障碍用户的操作习惯和隐私需求，既提供了深度个性化设置，又保持了界面简洁性。在实现时建议采用渐进式披露原则，将复杂功能隐藏在二级界面中，主界面保持清晰简洁。
